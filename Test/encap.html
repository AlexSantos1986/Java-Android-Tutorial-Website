Understand Encapsulation principles

Encapsulation is a concept of storing data together with methods that operate on that data. Objects are used as the container for the data and code.I will be showing the principles of encapsulation and how it should be applied by a developer. Encapsulation allows for data and method hiding. Information hiding makes it possible to expose a public interface while hiding the implementation details. I will also show how some good basic design principle that should be used with the java language.


Good design with Encapsulation

The fundamental of an object-oriented language is that software is designed by creating discrete objects that interact to make up the functionality of the application.Encapsulation is the concept of storing similar data and methods together in discrete classes. In many non-object-oriented languages, there is no association between where the data is and where the code is.This can increase complexity of maintaining the code, because ofter the variables that the code is using are spread apart over the code base. Bugs in the code can be hard to find and resolve due to different remote procedures using the same variable.
Encapsulation tries to solve this problems.It let you create code that is easier to read and maintain by allowing you to group related variable and methods together in classes.



//save as Student.java
package com.MAD;
public class Student{
private String name;

public String getName(){
return name;

}
public void setName(String name){
this.name=name;
}
}

Access Modifiers

Access modifiers are keywords that define what can access methods and instance variables.The three access modifier are private, protected, and public. These all change the default level of access.The default access level does not use a keyword and is assigned to a method or instance variable when neither private, protected or public uses and the area is left blank.Access modifiers are an important concept in object-oriented language.They allow the implementation details to be hidden in a class.The developer can choose specially what parts of class accessible to other objects.

As mentioned, Java uses three access modifiers: private, protected and public. The private access modifier is the most restrictive and most commonly used access modifier.Any method or instance variable that is marked as private can be accessed only by other methods in the same class. Subclasses cannot access instance variable or methods that are private. The following is an example of the private keyword:


private int numberOfStudents;

private int calculateAverage() {...}

The default access level is the second most restrictive.It is often referred to as package-private.This access level allows to its methods and instance variables from code that is the same package.The default access level does not have a keyword to indicate it is use.A method or instance variable is set to default when an access modifier is omitted.The following is an example of this access level in use:


int Student;

double calculateStudentGrade(){...}

The protected access modifier is the third most restrictive.It is the same as the default access level but adds the ability of subclasses outside of the package to access its methods or instance variable.This means the methods that can access this data must either be in the same package or be in a subclass of the class that contains the protected data. Remember that a subclass is a class is a class that extends another class. Have a look at the example of the use of protected modifier:


protected String studentName;

protected char getStudentFirstChar(){...}

The final access modifier is public.This is the least restrictive and second most common access modifier.The public access modifier provides no restriction to what can access its methods and instance variable regardless of which package it is contained in or which superClass it extends. A variable named marked as public is accessible to the world.The following is an example of using the public access modifier:


public String StudentAddress;

public int findZipCode;

Getters and Setters

Getters and setters are the basic final concept of hiding information and encapsulation.As was discussed previously,it is good design to make all instance variable private.This means that subclasses or any external classes have no way to access these variables. Sometimes an external object may need to read one of these variables to determine its state, or it may have to set the value.To achieve this, a public method is created for the variable, both to get abd set value. These are called getters and setters.They can be as simple as one line that only sets or returs a value. The example below is a class that has one instance variable and a setter and getter:


public class MyClass{

private String className;

public String getClassName(){

return className;

}

public void setClassName(String className){

this.className= className;

}

}

The two methods that are present are getter and setter for the variable className.In this case,the class is giving read and write access to the variable via the methods. In some cases, a class may give only one or the other. The getter and setter in this demonstration are simple and only set or return the value.However ,if the class had to perform an action every time the className variable was changed,it could be done from the setter. For example, each time the className variable is changed,the class must record it.This can be done in the setter.This is the benefit of keeping instance variable private.It gives control to the class as to how its instance variable are accessed.

When developers are working with a class they expect to find getters and setters. They also expect name convention. When creating getter,the name should start with lowercase get, followed by variable name with no space and the first letter capitalized. When creating a setter, a similar convention should be followed. A setter should start with the word set, followed by the variable name with the first letter capitalized.
